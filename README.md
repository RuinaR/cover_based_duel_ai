# cover_based_duel_ai

엄폐물을 활용하는 1:1 전투 상황을 가정하고,  
에이전트가 관측값을 바탕으로 행동을 선택하도록 구성한 강화학습 기반 실험 프로젝트입니다.

단순히 학습 코드만 작성한 것이 아니라,  
**학습 환경**, **정책 설정**, **실행 모드 분리**, **시각화 실행**, **체크포인트 저장/로드**까지 포함한  
프로젝트 형태로 구성했습니다.

---

## 프로젝트 소개

이 프로젝트는 엄폐 기반 전투 상황을 단순화한 환경에서,  
에이전트가 위치/거리/상태 등의 관측값을 기반으로 행동을 선택하고  
그 결과에 따라 보상을 받아 학습하는 구조를 구현한 프로젝트입니다.

학습만 수행하는 스크립트에 그치지 않고,  
실행용 애플리케이션 구조를 분리하여 **train 모드**와 **visual 모드**를 따로 두었고,  
환경과 정책 생성, 실행 로직, 시각화 처리까지 나누어 관리했습니다.

---

### 주요 구성

- **ai**  
  정책 생성, 체크포인트 설정, 학습 관련 구성 담당

- **app**  
  실행 진입부, 학습 실행기, 시각화 실행기 등 애플리케이션 레벨 로직 담당

- **engine**  
  게임 루프 및 공통 실행 흐름 담당

- **models**  
  학습 모델 및 저장 데이터 관련 영역

- **shooter**  
  전투/환경 관련 구성 요소 담당

## 주요 기능

- train 모드 / visual 모드 분리
- 정책 설정 및 팩토리 구조 분리

---

## 실행 방식

### 1. train 모드
에이전트가 환경과 상호작용하며 보상을 통해 학습합니다.

### 2. visual 모드
학습된 결과 또는 현재 정책 기반 동작을 시각적으로 확인할 수 있습니다.

---

## 구현하면서 중점적으로 고민한 점

### 1. 실행 모드 분리
하나의 스크립트에 모든 로직을 몰아넣기보다  
`train`과 `visual` 모드를 나누어 학습과 시각화 목적을 분리했습니다.  
덕분에 유지보수성과 테스트 편의성이 좋아졌습니다.

### 2. 환경/정책/애플리케이션 레벨 코드 분리
강화학습 프로젝트는 환경, 정책, 실행 루프가 쉽게 뒤섞이기 때문에  
각 역할을 폴더와 파일 단위로 나누어 프로젝트 형태로 정리하려고 했습니다.

### 3. 체크포인트와 반복 실행 고려
실험성 코드에 그치지 않고,  
학습 중간 결과를 저장하고 다시 불러올 수 있도록 체크포인트 흐름을 두어  
반복 실험이 가능하도록 구성했습니다.

## 어려웠던 점

- 관측값, 행동, 보상 구조를 어떻게 설계해야 원하는 학습 방향이 나오는지 조정하는 과정이 어려웠습니다.
- 전투 상황을 단순화하면서도 학습 가능한 구조를 유지하는 데 고민이 많았습니다.
- 학습 로직과 시각화 로직이 섞이지 않도록 구조를 유지하는 것이 쉽지 않았습니다.

## 아쉬웠던 점

- 프로젝트 설명 문서가 부족해 처음 보는 사람이 환경 구성과 보상 설계를 빠르게 이해하기 어렵습니다.
- `.venv`, `.vscode`, `__pycache__` 등 개발 환경 관련 파일 정리가 더 필요합니다.
- 학습 결과에 대한 정량적 비교나 실험 기록을 더 체계적으로 남기지 못했습니다.

## 배운 점

이 프로젝트를 통해 강화학습에서 단순히 알고리즘만 아는 것이 아니라,  
**환경 설계, 관측값 구성, 보상 설계, 실행 구조 분리**가 얼마나 중요한지 배울 수 있었습니다.

또한 Python 프로젝트도 단일 파일이 아니라  
실행 구조를 나누고 유지보수 가능한 형태로 만드는 경험을 할 수 있었고,  
게임 AI/시뮬레이션과 연결되는 방향으로 사고하는 데 도움이 되었습니다.
